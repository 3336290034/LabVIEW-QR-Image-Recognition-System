## ProcessImageData 花屏 + 卡顿修复

### 花屏根因：转置维度写反
- LabVIEW 2D 数组是列优先(column-major)，内存中连续 height 个像素 = 一列数据
- 正确做法：`cv::Mat(width, height, CV_8UC4, imageData)` 按行读取 → 每行正好是一列 → transpose 得到正确的 height×width
- 之前错误地用 `cv::Mat(height, width, ...)` 导致行列数据混读
- 修复：改为 `cv::Mat(width, height, CV_8UC4, imageData)` + transpose

### 卡顿根因：多策略多尺度扫描
- ProcessImageData 摄像头 30fps 场景，原代码走 processFrame（5策略×5尺度=最多25次ZBar扫描）
- 新增 processFrameFast 方法：仅灰度转换 + 1x/2x 两个尺度扫描，耗时约 20-50ms
- ProcessImageData 调用 processFrameFast，ProcessImageFile 保持 processFrame 多策略不变

### 编译
- DLL 编译成功，7个导出函数全部正确

### 图像方向修正
- transpose 后图像顺时针偏了约90度
- 尝试 ROTATE_90_COUNTERCLOCKWISE → 逆时针过头
- 尝试 ROTATE_90_CLOCKWISE（无transpose）→ 上下颠倒
- 最终方案：Mat(width, height) + transpose + ROTATE_90_CLOCKWISE
- 用户确认需要"再顺时针90度"修正，已编译待测试
